# 都市异能核心人物创建

## 设计理念

**拒绝套路，追求真实感。** 每个角色都应该像一个真实的人——有矛盾的性格、有说不出口的软肋、有让读者又爱又恨的缺点。好角色不是"完美"的，而是"真实"的。

核心原则：
- **反差感**：每个角色至少有一个与外在形象矛盾的内在特质
- **记忆点**：每个角色必须有一个独特的口癖/习惯/怪癖，让读者一提就能想起来
- **灰度感**：没有纯粹的好人和坏人，只有不同立场下的选择
- **趣味性**：角色团队中必须有搞笑担当和中二担当，调节叙事节奏

## 核心人物定义

核心人物 = 贯穿全书的主线角色，从第一弧就开始埋线，最后一弧仍然有戏份。

**只创建以下角色，不多不少：**

| 类型 | 数量 | 说明 |
|------|------|------|
| protagonist | 1 | 主角——有深度的复杂人物，不是模板化爽文主角 |
| female_lead | 1-2 | 主要女主——有独立主线的角色，不是感情附属品 |
| antagonist | 1 | 终极对手——立场对立但动机自洽的镜像角色 |
| supporting | 3-4 | 核心团队——每人承担不同叙事功能（搞笑/中二/引路/情感锚点） |

**总计 6-8 个角色**，不要创建弧线特有的炮灰和中级反派。

## 主角设计要求

**拒绝"龙傲天"和"舔狗逆袭"的老套路。**

- **性格层次**：至少有两面性——比如表面吊儿郎当实际心思缜密，或者嘴上刻薄内心柔软。性格中要有明显的**缺陷**（傲慢/偏执/不善表达/过度自我保护），这个缺陷会在全书中逐步成长
- **内心独白**：要有丰富的内心戏，偶尔吐槽、偶尔自嘲、偶尔中二上头，让读者觉得"这人跟我一样"
- **初始境遇**：不一定是被退婚被踩，可以是自己选择的困境（为了某个秘密主动隐忍/为了保护某人故意低调），要有**主动性**而不是纯被动挨打
- **核心驱动**：驱动力要有层次感——表面目标（变强/报仇）之下要有更深层的心理需求（被认可/不再失去/弥补遗憾）
- **禁忌**：不能全知全能、不能每次都做出最优解、不能对所有人都一样好。要有偏心、有判断失误、有说错话的时候

## 女主设计要求

**女主是独立角色，不是主角的奖励。**

- **独立主线**：女主必须有自己的目标和困境，这条线与主角时而交汇时而分离，而不是全程围绕主角转
- **性格鲜明**：可以是毒舌学霸、社恐天才、表面温柔实则腹黑——但绝不能是"善良、温柔、美丽"这种空洞描述
- **与主角的化学反应**：两人之间要有**张力**——价值观冲突/性格互补/互相看不顺眼但又不得不合作。感情是在碰撞中自然生长的，不是一见钟情
- **实力定位**：女主在某个领域必须强于主角，形成互补而非依附关系
- **禁忌**：不能动不动脸红、不能遇到危险就等主角救、不能没有主角就活不下去

## 终极对手设计要求（不叫"大Boss"，叫"对手"）

**最好的反派是"如果换个立场，读者会支持他"的那种角色。**

- **动机自洽**：对手不是"为恶而恶"，而是在自己的逻辑框架内做出了合理选择。他的目标甚至可能是正义的（拯救族人/维护秩序/阻止灾难），只是手段偏激或牺牲了不该牺牲的人
- **镜像关系**：对手与主角应该有某种对称性——相似的天赋、相似的经历，但在关键分岔路口做了不同选择。读者要能感受到"主角如果走错一步，可能就变成了他"
- **人格魅力**：对手要有令人欣赏的一面——信守承诺/爱护下属/有自己的底线。不能是一个让人纯粹厌恶的符号
- **与主角的互动**：两人之间除了对抗，还要有惺惺相惜或者理念交锋的戏份。最好的对决不是打斗，而是两个人都说服不了对方
- **前期布局**：前期通过蛛丝马迹暗示其存在和影响力，但不直接出场

## 核心配角设计要求

每个配角必须承担**独特的叙事功能**，不能功能重叠：

- **搞笑担当**（必须有）：负责调节气氛、制造笑点。但搞笑不是全部——在关键时刻要有反差的严肃和可靠。比如平时没正经但保护朋友时最拼命的那种人。可以有独特的口癖或行为怪癖
- **中二担当**（必须有）：说话夸张、给自己取帅气的称号、做事浮夸但意外有效。这个角色是读者的快乐源泉，但中二外壳下要藏着认真的内核——也许中二是他应对压力的方式
- **引路人/前辈**：不是无敌的老爷爷，而是有自己遗憾和局限的过来人。教导方式不能是"我把毕生功力传给你"，而是点拨思路、提出问题让主角自己悟。这个角色自身也有未完成的执念
- **情感锚点**：主角最在乎的人。不一定是长辈，可以是妹妹/发小/欠了人情的故人。这个角色的存在感不靠戏份多，而靠"主角提到这个人时的态度变化"

**配角禁忌**：
- 不能所有配角都崇拜主角
- 配角之间要有独立于主角的关系网（互相拌嘴/暗恋/竞争/师兄弟）
- 每个配角至少有一个"主角不知道的秘密"

## 角色关系网络要求

角色之间的关系不能只有"主角→其他人"的单向辐射，要形成**网状结构**：
- 至少有两组角色之间存在独立于主角的关系或矛盾
- 对手阵营和主角阵营之间要有灰色地带的人物关联（前同门/旧识/血缘）
- 角色之间的关系要有**变化空间**（初期敌对→后期理解，初期信任→后期背叛的可能性）

## 输出格式

输出 JSON 数组，每个人物包含：

```typescript
{
  name: string,
  role: "protagonist" | "female_lead" | "antagonist" | "supporting",
  age: number,
  appearance: string,           // 不要只写"帅"或"美"，要有辨识度的细节（疤痕/习惯性动作/标志性穿搭）
  personality: string,          // 写出至少两面性，包含性格缺陷
  innerConflict: string,        // 内心矛盾/未解决的心结
  quirk: string,                // 口癖/怪癖/标志性行为——让读者记住的记忆点
  background: string,
  abilities: string,
  currentRealm: string,
  relationshipToProtagonist: string,
  narrativeFunction: string,    // 在故事中承担的叙事功能（搞笑担当/中二担当/情感锚点/理念对照等）
  hiddenSecret: string,        // 其他角色不知道的秘密，用于后续剧情埋线
  firstAppearance: number  // 预计首次出现的章节号
}
```

**严格控制数量：6-8 个核心角色。弧线反派和炮灰将在大纲确定后按弧创建。**
