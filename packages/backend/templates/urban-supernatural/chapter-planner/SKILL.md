# 章节规划

## 设计理念

**章节是读者体验的最小单位，每一章都必须让人"想翻下一页"。**

好的章节不是靠打脸爽感吊着读者，而是靠**悬念的层层递进、角色的真实碰撞、信息的精准释放**让人停不下来。每一章结束时，读者脑中应该有一个没解决的问题。

核心原则：
- **信息控制**：每章精确控制"透露什么、隐藏什么、暗示什么"——信息差是张力的核心引擎
- **角色驱动**：情节由角色的性格、矛盾和选择推动，而不是外部事件强行推进
- **节拍有呼吸**：紧张场景后要有喘息，高燃之后要有沉淀，不能一路高潮到底
- **伏笔意识**：每章都要考虑——这里能自然地埋入什么？之前埋的哪个伏笔可以在这里回收？

## 每章结构模板

```
开头（200-300字）：承接上章情绪，快速建立本章方向
中段（1800-2000字）：核心情节推进，包含至少一个角色内心的变化
结尾（200-300字）：制造"未完成感"——悬念、反转或情感余韵
```

## 章节节拍（beats）设计原则

每章必须有 3-5 个清晰的情节节拍。节拍不是机械的"事件列表"，而是**情绪的起伏曲线**。

### 节拍类型

| 类型 | 说明 | 何时使用 |
|------|------|----------|
| 铺垫 | 看似日常的场景中埋入不对劲的细节 | 弧线前期，为后续反转做准备 |
| 碰撞 | 角色之间因立场/性格/信息差产生正面冲突 | 需要推进关系或暴露矛盾时 |
| 抉择 | 主角面对两难选择，每个选项都有代价 | 展现角色底色，推动成长 |
| 揭示 | 某个隐藏信息被暴露，改变角色或读者的认知 | 制造"卧槽"感 |
| 沉淀 | 紧张事件后的安静时刻，角色消化发生的事 | 高潮后的情感回收 |
| 暗涌 | 表面平静但暗藏危机的场景 | 制造不安感 |

### 节拍设计示例

**例子（困局反转章节）**：
1. 铺垫：主角根据已有信息制定计划，看似胸有成竹
2. 碰撞：计划执行中遭遇预料外的变数——信息有误/对手另有后手
3. 抉择：两条路都不完美，主角必须在有限时间内做出取舍
4. 揭示：靠之前不起眼的细节（伏笔回收）找到第三条路
5. 沉淀：危机解除，但主角意识到自己的某个认知是错的

**例子（角色关系深化章节）**：
1. 铺垫：配角之间的独立互动（不经过主角），展现各自的立场
2. 暗涌：某个角色的反常行为引起另一角色的注意
3. 碰撞：两人就某件事产生分歧，各自的"隐藏秘密"影响了判断
4. 揭示：一方说出了某句触及对方心结的话，关系发生微妙变化
5. 沉淀：独处时的内心独白——想说但没说出口的话

**例子（认知颠覆章节）**：
1. 铺垫：主角按照既有认知行动，一切看似正常
2. 暗涌：某个细节与之前掌握的信息矛盾，但主角没注意（读者可能注意到）
3. 碰撞：矛盾激化，主角不得不正视"不对劲"的地方
4. 揭示：真相与主角以为的完全不同——曾经信任的人/规则/记忆出了问题
5. 抉择：面对新真相，主角必须决定"接下来怎么办"

## 结尾钩子设计

**结尾钩子不是为了"骗点击"，而是让读者带着一个"未解决的问题"入睡。**

### 钩子类型

| 类型 | 说明 | 效果 |
|------|------|------|
| 信息暗示 | 某个角色得到了读者（或主角）还不知道的信息 | 信息差制造焦虑 |
| 选择悬置 | 主角面临抉择，本章在做出决定前结束 | 读者代入思考 |
| 视角切换 | 切到对手/配角视角，揭示主角不知道的暗流 | 拓宽格局 |
| 异常信号 | 一个不起眼但"不对劲"的细节 | 制造不安感 |
| 情感余震 | 重大事件后，角色独处时的一个微表情/动作 | 情感延续 |
| 伏笔触发 | 之前埋下的某个细节突然出现呼应 | "原来如此"感 |

### beats 最后一项与 endingHook 必须一致（重要）

- **`endingHook` 是本章收束的唯一真源**：正文最后一幕、最后一句台词只能来自这里（或与其字面一致）。
- **最后一个 beat** 只能写「如何逼近这一幕」，不得在 beat 里再指定**另一句**终场台词或另一套终场画面；若需要台词示例，必须与 `endingHook` 同一句，或只描述动作不写死台词。
- 规划时先定 `endingHook`，再写最后一 beat，避免出现「节拍里一句金句、钩子里又一句近义金句」导致写作模型拼出双结尾。

### endingHook 写法规范（重要）

`endingHook` 必须是**可直接书写的最后一幕或最后一句话**，不是对主题意图的描述。

| 错误写法（主题描述）| 正确写法（具体场景/台词）|
|---|---|
| "系统发布任务，将主角推向危险道路" | "林逸低声自语：'吞噬……更多么？'脚步声散入夜风" |
| "反派察觉到主角的能力，准备追杀" | "便利店的灯光里，男人缓缓掏出手机，按下一个号码" |
| "主角完成突破，实力大增" | "他握紧拳头——骨节轻轻炸响，连他自己都吓了一跳" |
| "两人关系出现裂痕" | "她转身走进雨里，没有回头。他张了张嘴，什么也没说" |

`endingHook` 的三个标准：
1. **具体可写**：读者能看到这个画面，而不是"感受到一种意图"
2. **自然停顿点**：写完这句话后，章节在此结束完全合理
3. **50字以内**：越短越有力，不要写成一段话

## 节奏控制

### 微观节奏（每 3-5 章一个循环）

```
铺垫/日常 → 矛盾触发 → 紧张对峙 → 突破/反转 → 短暂喘息
```

在每个循环中：
- **至少 1 章**包含角色之间的非战斗互动（日常、对话、内心戏）
- **至少 1 章**推进主线之外的副线（配角关系、感情线、世界观扩展）
- **高潮章节后**必须安排 1 章"沉淀"——角色消化刚发生的事，读者感受情感余韵

### 张力管理

- **连续紧张章节**：不超过 3 章，之后必须有喘息/日常/角色互动调剂
- **能力突破**：不能机械化安排，必须绑定关键剧情节点。突破来自"经历特定事件后的领悟"，且伴随代价或副作用
- **感情线**：不是进度条式推进，而是在冲突、合作、误解、坦诚中波浪式前进。最好的感情戏发生在生死抉择中、理念冲突中、共同面对危机中
- **信息释放**：每 3-5 章释放一个新信息点（关于世界真相/角色秘密/势力格局），保持读者的求知欲

## 角色运用规则

### 叙事功能调度

根据核心角色的叙事功能合理调度出场：
- **搞笑担当**：紧张弧段后安排出场调节气氛，但在关键时刻展现反差的严肃面
- **中二担当**：需要活跃气氛或打破僵局时出场，中二外壳下偶尔露出认真内核
- **引路人/前辈**：主角遇到认知瓶颈时出场点拨，但不给答案——提出问题让主角自己悟
- **情感锚点**：在主角面临重大抉择或情绪低谷时，通过回忆/出场/被提及来影响主角的选择

### 角色互动要求

- 每 5 章内至少有 1 场**不经过主角的配角互动**——展现角色之间独立的关系网
- 主角的内心独白要自然——可以吐槽、自嘲、中二上头，体现真实性格
- 角色的"隐藏秘密"要通过行为细节暗示，而不是直接告诉读者
- 角色的性格缺陷要在情节中自然体现——不是刻意降智，而是"这个人就是会在这种事上犯错"

## 伏笔追踪

每章规划必须考虑伏笔的埋入和回收：

- **foreshadowPlanted**：本章自然埋入的伏笔（可选），必须是看似随意的细节——不能刻意到让读者觉得"这里肯定有伏笔"
- **foreshadowHarvested**：本章回收的之前伏笔（可选），回收时要让读者有"原来如此"的满足感

伏笔类型：
- **台词伏笔**：某个角色说了一句看似无心的话，后来证明大有深意
- **物件伏笔**：某个不起眼的物品/场景在后面的关键时刻发挥作用
- **行为伏笔**：某个角色的反常举动，后来揭示是因为他知道某个秘密
- **信息伏笔**：故事中提到的某个"常识"或"规则"，后来发现有例外

## 输出格式

```json
[
  {
    "chapterNumber": 1,
    "title": "第一章 章节标题",
    "pov": "主视角角色名",
    "location": "场景地点",
    "summary": "本章一句话概括（包含核心矛盾或变化）",
    "beats": [
      "铺垫：承接上章，……",
      "碰撞：……",
      "抉择/揭示：……",
      "高潮：……",
      "沉淀/悬念：[与endingHook一致的具体描述]"
    ],
    "tensionType": "困局反转 | 认知颠覆 | 关系深化 | 势力博弈 | 成长蜕变 | 日常暗涌",
    "characterArcs": {
      "角色名": "本章中该角色的内在变化或状态（一句话）"
    },
    "foreshadowPlanted": "本章埋入的伏笔（简述内容，如无则为null）",
    "foreshadowHarvested": "本章回收的伏笔（引用来源章节，如无则为null）",
    "endingHook": "本章最后一句话或最后一个镜头的精确描述（50字以内，具体可写）",
    "wordTarget": 2500
  }
]
```

### 字段说明

- **tensionType**：本章的核心张力类型，用于检查节奏多样性——连续章节不应重复同一种张力类型
- **characterArcs**：追踪本章中有戏份的角色状态变化，确保角色弧光持续推进
- **foreshadowPlanted / foreshadowHarvested**：伏笔追踪，与大纲中的伏笔系统对应

### 节奏自检清单

规划完一批章节后，用以下标准自检：
- [ ] 连续紧张章节是否超过 3 章？
- [ ] 每 3-5 章是否有一个信息释放点？
- [ ] 是否有配角之间独立于主角的互动？
- [ ] tensionType 是否足够多样（不连续重复）？
- [ ] 伏笔是否有埋有收，且分布合理？
- [ ] 角色的出场是否符合其叙事功能？
- [ ] 高潮章节后是否安排了情感沉淀？
