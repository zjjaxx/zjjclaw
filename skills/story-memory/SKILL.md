# 故事记忆追踪

## 职责

读取最新章节内容，更新并维护完整的故事状态记录，确保后续写作的一致性。

## 需要追踪的核心状态

### 1. 主角状态
- 当前境界（精确到小境界）
- 已掌握的能力列表
- 当前目标（近期的，3-5章内）
- 当前位置
- 与各人物的关系状态（恶化/改善/新建立）

### 2. 打脸队列管理

**待打脸**（关键！不能让反派逍遥太久）：
- 记录每个反派的恶行章节
- 距离被打脸不超过 20 章
- 按优先级排列（最近的恶行最先处理）

**已打脸记录**：
- 对方被打脸的具体方式
- 打脸章节
- 打脸后对方的状态（逃跑/臣服/消失）

### 3. 情节线状态

每条情节线需标注：
- active（正在进行）
- resolved（已解决）
- pending（埋下伏笔，未到时机）

**重要**：pending 的伏笔不要超过 10 条，否则后续无法收尾。

### 4. 感情线进度

每个女性角色的感情阶段：
```
陌生 → 注意到 → 有好感 → 暧昧 → 表白 → 确认关系 → 深化
```

### 5. 势力关系图

记录主角与各势力的关系变化：
- 友好/中立/敌对
- 欠了谁的人情，谁欠了主角的人情

## 更新规则

每章写完后必须检查：
- [ ] 主角境界有无变化？
- [ ] 有无新增待打脸？
- [ ] 有无情节线新开/关闭？
- [ ] 感情线有无推进？
- [ ] 有无新埋下的伏笔？
- [ ] recent_events 更新（保留最新5条）
