# 章节规划

## 每章结构模板

```
开头（200-300字）：承接上章悬念，迅速解决
中段（1800-2000字）：本章核心情节推进
结尾（200-300字）：新的悬念/爽点，让读者追更
```

## 章节节拍（beats）设计原则

每章必须有 3-5 个清晰的情节节拍：

**例子（打脸章节）**：
1. 反派当众羞辱主角
2. 主角沉默，周围人等着看笑话
3. 主角展示某个证明（实力/身份/资源）
4. 反派懵逼/惊恐
5. 反转完成，主角离开，留下场面

**例子（修炼突破章节）**：
1. 主角受到某个触发（危险/资源/顿悟）
2. 进入修炼/突破状态描写
3. 阻碍出现（资源不足/敌人打扰）
4. 克服阻碍（金手指帮助）
5. 成功突破，实力提升描写

## 结尾钩子类型

| 类型 | 模板 | 效果 |
|------|------|------|
| 悬念型 | "就在这时，门突然被踢开了。" | 让读者追看下一章 |
| 反转型 | "他以为结束了，却没想到……" | 爽后再爽 |
| 危机型 | "情报显示，那人已经出关了。" | 制造紧张感 |
| 爽感型 | "此刻，整个宴会厅鸦雀无声。" | 享受被仰望的感觉 |
| 升级型 | "面板上，那个数字缓缓跳动。" | 期待下一步成长 |

## 节奏控制

- **连续打脸章节**：不超过3章，之后要有修炼/感情/悬疑调剂
- **修炼突破**：不能每章都突破，2-5章铺垫一次突破
- **感情线**：每10章推进一小步，不能喧宾夺主

## 输出格式

```json
[
  {
    "chapterNumber": 1,
    "title": "第一章 章节标题",
    "pov": "林凡",
    "location": "华都一中",
    "summary": "本章一句话概括",
    "beats": [
      "开头：承接上章，...",
      "中段1：...",
      "中段2：...",
      "高潮：...",
      "结尾：..."
    ],
    "faceSlapMoment": "（如有）描述打脸情节",
    "breakthroughMoment": "（如有）描述突破情节",
    "romanceMoment": "（如有）描述感情线推进",
    "endingHook": "结尾留下的悬念/钩子",
    "wordTarget": 2500
  }
]
```
