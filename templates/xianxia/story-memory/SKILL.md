# 玄幻修真故事记忆追踪

## 追踪的核心目的

防止以下低级错误：
- 主角在前章刚突破到金丹，下一章突然又是筑基境界
- 反派在第10章受重伤，第15章若无其事地出现且实力未降
- 埋下的法宝/功法伏笔被遗忘，读者追着问"那本上古功法呢"
- 女主对主角从警惕变成信任，但下一章又恢复警惕（无故回退）

---

## 需要追踪的状态

### 主角状态
- 当前境界（精确到前期/中期/后期/圆满）
- 已掌握功法（名称+品阶+熟练度描述）
- 持有法宝（名称+品阶+特殊效果）
- 当前修炼目标（主线目标）
- 当前位置（具体地点）
- 受伤/状态异常（是否带伤，何时痊愈）

### 势力关系网络
- 宗门内的盟友（弟子/长辈）
- 宗门内的敌对者（已打脸/未打脸）
- 宗门外的势力关系（盟友/敌人/中立）

### 打脸账本（最关键）
- 待打脸对象：**谁**，因**何事**，在**第几章**触犯
- 已打脸记录：打脸方式，读者观众反应
- 规则：**单个打脸对象悬挂不得超过20章**（否则读者遗忘）

### 活跃剧情线
- 已揭开的身世/金手指谜题（已知几成？）
- 悬挂中的秘境/拍卖/比武（预计何章解决）
- 已埋下但未触发的伏笔（上限10条）

### 情感进度
- 各女主与主角的关系阶段
- 最近一次有意义的互动（第几章，发生了什么）

---

## 更新规则

每章写完后必须更新：
1. 若主角境界发生变化 → 更新境界和修炼相关字段
2. 若打脸发生 → 将对应条目从 pendingFaceSlaps 移入 resolvedFaceSlaps
3. 若新反派行为触犯主角 → 加入 pendingFaceSlaps
4. 若情感线有推进 → 更新 romanceProgress
5. 若新伏笔埋入 → 加入 foreshadowing（超10条须先解决旧伏笔）
6. 必须更新 recentEvents（保留最近3-5章关键事件摘要）
